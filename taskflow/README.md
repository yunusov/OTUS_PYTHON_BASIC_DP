# TaskFlow – корпоративная система управления задачами и проектами

**TaskFlow** – это веб-приложение, созданное для эффективного управления проектами, задачами и командным взаимодействием. Проект разработан на Python с использованием асинхронного стека и демонстрирует чистую архитектуру, тестируемость и готовность к развертыванию в контейнерах.

---

## 📌 О проекте

TaskFlow решает повседневные задачи командной работы:
- создание проектов и постановка задач;
- назначение ответственных, контроль статусов и приоритетов;
- обсуждение задач через комментарии;
- быстрый поиск и фильтрация;
- разграничение доступа на основе сессионной аутентификации.

Приложение написано с акцентом на качество кода, модульность и расширяемость.

---

## 🚀 Ключевые возможности

- **Аутентификация и управление пользователями**  
  Регистрация, вход, выход, защищённые маршруты через сессии (HTTP-only cookies).

- **Проекты**  
  Создание, редактирование, удаление, просмотр списка и детальной страницы проекта. Владелец имеет полный контроль над проектом.

- **Задачи**  
  Каждая задача содержит заголовок, описание, статус (*To Do*, *In Progress*, *Done*), приоритет (*Low*, *Medium*, *High*, *Critical*), дедлайн, исполнителя и автора.  
  – Быстрое изменение статуса по кнопке.  
  – Фильтрация по статусу, приоритету, исполнителю.  
  – Поиск по названию и описанию.

- **Комментарии**  
  Участники проекта могут оставлять комментарии к задачам. Автор или владелец проекта может редактировать/удалять комментарии.

- **Адаптивный интерфейс**  
  На основе Bootstrap 5, поддерживает работу на десктопах, планшетах и смартфонах.

- **Готовность к развертыванию**  
  Приложение упаковано в Docker-контейнеры, настройка окружения выполняется через переменные окружения.

## ⚙️ Установка и запуск

1. **Клонируйте репозиторий**  
   ```bash
   git clone https://github.com/yunusov/OTUS_PYTHON_BASIC_DP.git
   cd taskflow

2. **В корне проекта выполните:**
    ```bash
    docker-compose up -d

Сервисы будут запущены:

- Приложение – на порту 8000 (внутри контейнера)

- PostgreSQL – на порту 5432

- MailDev (для перехвата писем) – веб-интерфейс на http://localhost:1080

После запуска приложение будет доступно по адресу http://localhost:8000.

## Переменные окружения (.env)

Создайте файл .env на основе .env.example:

### Приложение
- SECRET_KEY=your-secret-key-here
- DEBUG=False

### База данных
- DB_HOST=db
- DB_PORT=5432
- DB_USER=postgres
- DB_PASSWORD=postgres
- DB_NAME=taskflow

### SMTP для уведомлений (опционально, можно использовать MailDev)
- SMTP_HOST=maildev
- SMTP_PORT=1025
- SMTP_USER=
- SMTP_PASSWORD=
- SMTP_USE_TLS=False
